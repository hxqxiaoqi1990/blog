linux常用命令

命令大全

  1. ls 显示目录文件 
         选项：
     -d 显示当前目录
     -i    显示inode号
     -h    常见单位显示
     -a    显示所有文件包括隐藏文件
  2. cd 切换目录（shell内置命令）
         cd - 进入上一次目录
     cd ~ 进入家目录
     cd .. 进入上级目录
  3. pwd 显示当前路径
  4. mkdir 建立目录
         例：mkdir -p /huang/xiao/qi 递归建立
  5. rm 删除文件 
         选项：-r递归，删除目录 
     -f强制
     例：rm -rf 文件和目录
  6. tree 显示目录下所有目录树（要安装的命令）
  7. touch 创建新文件或修改文件时间
  8. cat 查看文件内容 
         选项：-n显示行号
     例：cat -n 文件名
  9. more 分页显示文件内容
         空格键：下翻        n键：上翻        q键：退出
  10. less 分行显示文件内容
        空格键或pgdn：下翻    pgup键：上翻 q键：退出
  11. head 显示文件头，默认10行
        选项：-n 指定显示文件头几行
  12. tail 显示文件后几行，默认10行
        选项：-n 指定显示文件头几行
    -f    监听文件尾部
  13. cp 复制文件（如果复制链接文件，则复制原文件）
        选项：-r 复制目录（包括目录内文件数据）
    -p 连带文件属性复制
    -d 若源文件是链接文件，则复制链接属性

    例：cp -pdr 源文件位置 目的位置

  89. dd用指定大小的块拷贝一个文件
        if=指定源文件
    of=指定目的文件
    bs=bytes：同时设置读入/输出的块大小为bytes个字节
    count=blocks：仅拷贝blocks个块，块大小等于ibs指定的字节
    dd应用实例
  90. 将本地的/dev/hdb整盘备份到/dev/hdd
          dd if=/dev/hdb of=/dev/hdd
  91. 将/dev/hdb全盘数据备份到指定路径的image文件
          dd if=/dev/hdb of=/root/image
  92. 将备份文件恢复到指定盘
          dd if=/root/image of=/dev/hdb
  93. 备份/dev/hdb全盘数据，并利用gzip工具进行压缩，保存到指定路径
          dd if=/dev/hdb | gzip > /root/image.gz
  94. 将压缩的备份文件恢复到指定盘
          gzip -dc /root/image.gz | dd of=/dev/hdb
  95. 备份与恢复MBR
          a.备份磁盘开始的512个字节大小的MBR信息到指定文件：
      dd if=/dev/hda of=/root/image count=1 bs=512
      count=1指仅拷贝一个块；bs=512指块大小为512个字节。
      b.恢复：
      dd if=/root/image of=/dev/had
      将备份的MBR信息写到磁盘开始部分
  96. 拷贝内存内容到硬盘
          dd if=/dev/mem of=/root/mem.bin bs=1024 (指定块大小为1k)
  97. 拷贝光盘内容到指定文件夹，并保存为cd.iso文件
          dd if=/dev/cdrom(sr0) of=/root/cd.iso
  98. 增加swap分区文件大小
          第一步：创建一个大小为256M的文件：
      dd if=/dev/zero of=/swapfile bs=1024 count=262144
      第二步：把这个文件变成swap文件：
      mkswap /swapfile
      第三步：启用这个swap文件：
      swapon /swapfile
      第四步：编辑/etc/fstab文件，使在每次开机时自动加载swap文件：
      /swapfile swap swap default 0 0
  99. 销毁磁盘数据
         dd if=/dev/urandom of=/dev/hda1
     注意：利用随机的数据填充硬盘，在某些必要的场合可以用来销毁数据。
  100. 测试硬盘的读写速度
         dd if=/dev/zero bs=1024 count=1000000 of=/root/1Gb.file
     dd if=/root/1Gb.file bs=64k | dd of=/dev/null
  101. 确定硬盘的最佳块大小：
         dd if=/dev/zero bs=1024 count=1000000 of=/root/1Gb.file
     dd if=/dev/zero bs=2048 count=500000 of=/root/1Gb.file
     dd if=/dev/zero bs=4096 count=250000 of=/root/1Gb.file
     dd if=/dev/zero bs=8192 count=125000 of=/root/1Gb.file

    lvextend -L +大小 分区名        #增加逻辑卷容量，只修改MBR表
    resize -f 分区名 分区实际大小            #修改superblock大小            e2fsck -f 分区名                #检查文件系统

  108. dd命令详解
        作用
    dd：用指定大小的块拷贝一个文件
    格式
    dd if=/dev/zero of=/指定目录 bs=1M count=1000
    if=文件名：输入文件名，缺省为标准输入。即指定源文件。< if=input file >
    of=文件名：输出文件名，缺省为标准输出。即指定目的文件。< of=output file >
    bs=bytes：同时设置读入/输出的块大小为bytes个字节。
    count=blocks：仅拷贝blocks个块，块大小等于ibs指定的字节数。
    dd应用实例
  109. 将本地的/dev/hdb整盘备份到/dev/hdd
         dd if=/dev/hdb of=/dev/hdd
  110. 将/dev/hdb全盘数据备份到指定路径的image文件
         dd if=/dev/hdb of=/root/image
  111. 将备份文件恢复到指定盘
         dd if=/root/image of=/dev/hdb
  112. 备份/dev/hdb全盘数据，并利用gzip工具进行压缩，保存到指定路径
         dd if=/dev/hdb | gzip > /root/image.gz
  113. 将压缩的备份文件恢复到指定盘
         gzip -dc /root/image.gz | dd of=/dev/hdb
  114. 备份与恢复MBR
         a.备份磁盘开始的512个字节大小的MBR信息到指定文件：
     dd if=/dev/hda of=/root/image count=1 bs=512
     count=1指仅拷贝一个块；bs=512指块大小为512个字节。
     b.恢复：
     dd if=/root/image of=/dev/had
     将备份的MBR信息写到磁盘开始部分
  115. 拷贝内存内容到硬盘
         dd if=/dev/mem of=/root/mem.bin bs=1024 (指定块大小为1k)
  116. 拷贝光盘内容到指定文件夹，并保存为cd.iso文件
         dd if=/dev/cdrom(sr0) of=/root/cd.iso
  117. 增加swap分区文件大小
         第一步：创建一个大小为256M的文件：
     dd if=/dev/zero of=/swapfile bs=1024 count=262144
     第二步：把这个文件变成swap文件：
     mkswap /swapfile
     第三步：启用这个swap文件：
     swapon /swapfile
     第四步：编辑/etc/fstab文件，使在每次开机时自动加载swap文件：
     /swapfile swap swap default 0 0
  118. 销毁磁盘数据
        dd if=/dev/urandom of=/dev/hda1
    注意：利用随机的数据填充硬盘，在某些必要的场合可以用来销毁数据。
  119. 测试硬盘的读写速度
        dd if=/dev/zero bs=1024 count=1000000 of=/root/1Gb.file
    dd if=/root/1Gb.file bs=64k | dd of=/dev/null
    通过以上两个命令输出的命令执行时间，可以计算出硬盘的读、写速度。
    /dev/zero，是一个输入设备，你可你用它来初始化文件。该设备无穷尽地提供0，可以使用任何你需要的数目——设备提供的要多的多。他可以用于向设备或文件写入字符串0。
    /dev/null——它是空设备，也称为位桶（bit bucket）。任何写入它的输出都会被抛弃。如果不想让消息以标准输出显示或写入文件，那么可以将消息重定向到位桶。外号叫无底洞，你可以向它输出任何数据，它通吃，并且不会撑着！
  120. 确定硬盘的最佳块大小：
        dd if=/dev/zero bs=1024 count=1000000 of=/root/1Gb.file
    dd if=/dev/zero bs=2048 count=500000 of=/root/1Gb.file
    dd if=/dev/zero bs=4096 count=250000 of=/root/1Gb.file
    dd if=/dev/zero bs=8192 count=125000 of=/root/1Gb.file
    通过比较以上命令输出中所显示的命令执行时间，即可确定系统最佳的块大小。
